1. What Is Lendora?

Lendora is a multi-vendor peer-to-peer rental marketplace built for Bangladesh. It lets verified sellers (vendors) list items for rent — cameras, tools, baby equipment, event gear — and verified customers (renters) book them with deposit, payment, and return tracking built in.

Platform Type

Multi-vendor rental marketplace (like Airbnb but for physical goods)

Target Market

Bangladesh — primarily Dhaka, with national coverage planned

Three User Types

  • Customer — browses & rents
  • Vendor — lists & manages items
  • Admin — platform oversight

Revenue Model

Platform takes 10% commission on each completed rental. Vendor receives 90% as payout.

Key Differentiators: Identity-verified vendors and renters (NID upload), deposit hold system, damage claim resolution, bKash & Nagad payment integration, real-time rental tracking through 10 status stages.

StageStatusWho ActsAPI CallWhat Happens in DB
1 PENDING_CONFIRMATION Customer creates booking POST /rentals Rental row created. Vendor receives notification.
2 CONFIRMED Vendor approves POST /rentals/:id/confirm Status updated. Customer receives notification.
3 READY_FOR_PICKUP Customer pays POST /rentals/:id/mark-ready Payment records created (RENTAL_PAYMENT + DEPOSIT). VendorPayout record created as PENDING. Vendor notified.
4 ACTIVE Vendor dispatches POST /rentals/:id/dispatch Status updated. Rental period begins. Customer notified.
5 RETURN_INITIATED Customer requests return POST /rentals/:id/initiate-return Status updated. Vendor notified.
6 COMPLETED Vendor confirms return POST /rentals/:id/confirm-return ReturnRecord created. If no damage: VendorPayout upgraded from PENDING → COMPLETED. VendorProfile.totalEarnings incremented. Customer notified.
Auto-completion: A background utility (utils/rental.utils.ts → autoCompleteExpiredRentals()) runs every time GET /rentals is called. It automatically moves any ACTIVE rental whose endDate has passed to COMPLETED.
Damage / Admin Review: If the vendor marks major damage during confirm-return, ReturnRecord.adminReviewRequired is set to true. The payout is NOT released automatically — admin must review and resolve the damage claim in the admin panel.

Commission Calculation

Formula used at rental creation: 
rentalFee      = pricePerDay × totalDays
platformFee    = rentalFee × platformFeeRate   (default: 10% = 0.10)
vendorPayout   = rentalFee - platformFee
deliveryFee    = product.deliveryFee (if delivery requested)
totalAmount    = rentalFee + depositAmount + deliveryFee
These values are stored in the rental row at creation time and never recalculated (except on extend).

How Authentication Works

Token Flow:
  1. User logs in → API returns accessToken (short-lived JWT, ~15min) in response body, and sets refreshToken (long-lived, ~7 days) in an HttpOnly cookie.
  2. Frontend stores accessToken in sessionStorage via Zustand auth store.
  3. Every API request includes Authorization: Bearer <accessToken>.
  4. When accessToken expires → Axios interceptor in lib/api.ts automatically calls POST /auth/refresh using the cookie, gets a new accessToken, and retries the original request.
  5. All pending requests during refresh are queued to avoid duplicate refresh calls.

16. How Every Calculation Works

Rental Fee Calculation (at booking creation)

// In: application/use-cases/rental/create-rental.use-case.ts

const totalDays      = Math.round((endDate - startDate) / 86_400_000)
const rentalFee      = product.pricePerDay × totalDays
const platformFeeRate = 0.10  // 10% commission
const platformFee    = rentalFee × platformFeeRate
const vendorPayout   = rentalFee - platformFee
const deliveryFee    = deliveryRequested ? product.deliveryFee : 0
const totalAmount    = rentalFee + product.depositAmount + deliveryFee

Extension Recalculation

// In: rental.routes.ts — POST /rentals/:id/extend

const newTotalDays    = Math.round((newEndDate - startDate) / 86_400_000)
const newRentalFee    = pricePerDay × newTotalDays
const newPlatformFee  = newRentalFee × platformFeeRate
const newVendorPayout = newRentalFee - newPlatformFee
const newTotalAmount  = newRentalFee + depositAmount + deliveryFee
// DB: rental updated with all new amounts

Deposit Deduction on Return

// In: confirm-return handler
// damageAmount provided by vendor during return confirmation

depositRefund    = max(0, depositAmount - damageAmount)
depositDeduction = min(depositAmount, damageAmount)
// Stored in: return_records.depositRefund and .depositDeduction

Late Return Penalty

// In: ConfirmReturnUseCase
lateDays    = days between expected endDate and actual return date
latePenalty = lateDays × (pricePerDay × 1.5)   // 150% daily rate
outstandingDue = max(0, latePenalty - depositAmount)

Vendor Average Rating

// In: review.repository.ts — after each new review
averageRating = AVG(reviews.rating) WHERE reviews.vendorId = vendorProfile.id
// Stored in: vendor_profiles.averageRating

Admin Analytics — Category Counts
